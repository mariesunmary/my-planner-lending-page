1. ЗАГАЛЬНА ІНФОРМАЦІЯ
Назва проєкту: FocusFlow
Мета: створити сучасний застосунок для планування часу, відстеження звичок і фінансів, який допомагає користувачам підтримувати баланс між роботою, навчанням і відпочинком.
Цільова аудиторія: студенти, фрілансери, молоді спеціалісти, які прагнуть структурувати свій день.
Основні конкуренти:
•	Notion
•	Todoist
•	Habitica
•	Google Calendar
•	TickTick
Унікальні особливості FocusFlow:
•	Інтерфейс підлаштований під спокійний стан — без зайвих нотифікацій, з приємною кольоровою палітрою.
•	Щоденні нагадування не про дедлайни, а про баланс: наприклад, “зроби паузу”, “час підбити підсумки дня”.
•	Об’єднання кількох функцій (звички, бюджет, проєкти, щотижневі цілі) в одному інтерфейсі.
•	Можливість аналітики прогресу у звичках і фінансах.
•	Система визначає, коли користувач найпродуктивніший, і підказує оптимальний час для звичок або задач.
•	Аналітика не просто показує цифри — вона формує інсайти: “ти найкраще працюєш у другій половині дня”. 2. СТРУКТУРА ПРОЄКТУ
2.1. Home
Мета модуля:
Знайомство користувача з додатком та створення приємного першого враження. Сторінка Home — це стартова точка, яка задає настрій роботи з FocusFlow і коротко пояснює його можливості.
Елементи інтерфейсу показано на Рисунку 1:
•	Привітальне повідомлення.
•	Короткий опис додатку та його основних функцій.
•	Меню для переходу до основних розділів (Habit Tracker, Monthly Budget, Project Planner, Weekly To-Do).
 
Рисунок 1 – Приблизний дизайн сторінки Home
2.2. Weekly To-Do
Мета: Організація щоденних завдань у межах поточного тижня. Сторінка дозволяє створювати, редагувати, видаляти та відмічати виконані завдання для кожного дня.
Елементи інтерфейсу:
1.	Шапка сторінки (header):
•	Назва розділу Weekly To-Do
•	Мотивуюча цитата
•	Поточний місяць і рік
2.	Опис сторінки: Короткий текст, який пояснює призначення модуля та підказує користувачу, як з ним працювати.
3.	Блок днів тижня:
•	Кожен день відображається як окрема форма.
•	Усередині — число та список завдань та кнопка для створення нового.
•	Якщо завдання є, поруч з ним з’являються кнопки редагування і видалення.
•	При кліку на завдання воно зачеркується (позначається як виконане).
4.	Кнопки дій:
•	Edit — відкриває поле для зміни тексту завдання.
•	Delete — видаляє завдання після підтвердження користувача.
•	Add Task – створює нове завдання.
Поля введення:
Task input — текстове поле для введення завдання (до 100 символів).
•	Тип: текст
•	Обов’язкове поле: так
•	Перевірка: не можна залишати порожнім.
Алгоритм роботи:
1.	Користувач вводить завдання у поле конкретного дня і натискає Enter або «Add Task».
2.	Завдання додається до списку, поруч зʼявляються кнопки Edit і Delete.
3.	При кліку на текст завдання воно зачеркується — вважається виконаним.
4.	Edit відкриває поле редагування, Delete видаляє, як показано на Рисунку 2.

2.3. Habit Tracker
Мета: Відстеження виконання щоденних звичок протягом місяця. Сторінка дозволяє додавати нові звички, відмічати дні їх виконання та наочно бачити свій прогрес у вигляді таблиці.
Елементи інтерфейсу:
1.	Шапка сторінки (header):
•	Назва розділу Habit Tracker
•	Мотивуюча цитата
•	Поточний місяць і рік
2.	Опис сторінки: Короткий текст, який пояснює призначення модуля та інструктує користувача, як користуватися трекером звичок.
3.	Навігація по місяцям:
•	По центру міститься назва місяця і 2 кнопки вперед та назад для перемикання трекера.
4.	Поле для додавання звички:
•	Містить текстове поле для введення назви звички та кнопку Add Habit.
5.	Таблиця трекера:
•	Верхній ряд містить числа днів місяця (1–31).
•	Ліва колонка містить список звичок, доданих користувачем.
•	Кожен перетин рядка (звичка) і стовпчика (день) — це клітинка, яку можна натиснути.
•	При натисканні клітинка зафарбовується синім кольором, що означає виконання звички у цей день.
6.	Кнопки дій:
•	Add Habit — створює нову звичку та додає її в таблицю.
•	Edit — відкриває поле для редагування назви звички
•	Delete Habit (за потреби, у майбутніх версіях) — видаляє звичку з таблиці.
Поля введення:
•	Habit input — текстове поле для введення назви звички (до 50 символів).
•	Тип: текст
•	Обов’язкове поле: так
•	Перевірка: не можна залишати порожнім.
Алгоритм роботи:
1.	Користувач вводить назву нової звички у поле та натискає Add Habit.
2.	Звичка додається у таблицю з новим рядком.
3.	Користувач відмічає виконані дні, натискаючи на клітинки відповідного рядка.
4.	Натиснута клітинка зафарбовується синім, що означає виконання звички.
5.	Дані зберігаються для подальшого перегляду прогресу, який видно на Рисунку 3.

2.4. Monthly Budget
Мета: Контроль і аналіз щомісячних витрат. Сторінка дозволяє додавати, редагувати, видаляти витрати та переглядати їх у таблиці для зручного фінансового планування.
Елементи інтерфейсу:
1.	Шапка сторінки (header):
•	Назва розділу Monthly Budget
•	Мотивуюча цитата
•	Поточний місяць і рік
2.	Опис сторінки: Короткий текст, який пояснює мету сторінки та допомагає користувачу зорієнтуватися, як додавати і переглядати витрати.
3.	Навігація по місяцям:
•	По центру міститься назва місяця і 2 кнопки вперед та назад для перемикання трекера.
4.	Форма введення витрат:
•	Розташована під навігацією.
•	Містить такі поля:
o	Назва витрати — короткий опис покупки або платежу
o	Кількість (сума) — числове значення витрати
o	Категорія — поле з можливістю самостійного введення або вибору з випадаючого списку (наприклад: Food, Transport, Entertainment, Other)
o	Дата — дата здійснення витрати
•	Кнопка Add Expense для додавання нового запису
5.	Таблиця витрат:
•	Структура таблиці:
o	Назва витрати
o	Кількість
o	Категорія
o	Дата
o	Дії (Actions) — кнопки Edit і Delete
Таблиця автоматично оновлюється після додавання або редагування запису.
6.	Кнопки дій:
•	Add Expense — додає новий запис
•	Edit — відкриває поля для редагування вибраного запису
•	Delete — видаляє витрату після підтвердження користувача
Поля введення:
•	Name (Назва витрати)
o	Тип: текст
o	Обов’язкове поле: так
o	Перевірка: не можна залишати порожнім
•	Amount (Кількість)
o	Тип: число
o	Обов’язкове поле: так
o	Перевірка: має бути більше 0
•	Category (Категорія)
o	Тип: текст або вибір із випадаючого списку
o	Обов’язкове поле: ні
•	Date (Дата)
o	Тип: дата
o	Обов’язкове поле: ні
Алгоритм роботи:
1.	Користувач заповнює поля форми та натискає Add Expense.
2.	Новий запис з’являється у таблиці витрат.
3.	Для кожного запису доступні кнопки Edit (редагувати) і Delete (видалити), які можна побачити на Рисунку 4.
4.	Після редагування або видалення таблиця автоматично оновлюється.
5.	Користувач може переглядати витрати за різні місяці, перемикаючись кнопками навігації.

2.5. Project Planner
Мета: Планування та організація проєктів із поділом на завдання. Сторінка дозволяє створювати нові проєкти, додавати до них завдання, змінювати статус виконання, редагувати та видаляти записи.
Елементи інтерфейсу:
1.	Шапка сторінки (header):
•	Назва розділу Project Planner
•	Мотивуюча цитата
•	Поточний місяць і рік
2.	Опис сторінки: Короткий текст, який пояснює призначення модуля та інструктує користувача, як створювати проєкти й додавати до них завдання.
3.	Форма додавання проєкту:
•	Розташована під описом сторінки
•	Містить поля:
o	Project Name — назва проєкту (обов’язкове поле)
o	Description — короткий опис (необов’язкове поле)
o	Deadline — кінцева дата виконання проєкту (необов’язкове поле)
Кнопка Add Project для створення нового проєкту
4.	Меню навігації по проєктах (sidebar):
•	Розташоване ліворуч
•	Містить список усіх створених проєктів
•	Клік по назві відкриває детальний перегляд вибраного проєкту
5.	Область планування проєкту (основний простір):
•	Відображається після вибору проєкту
•	Містить:
o	Назву вибраного проєкту
o	Опис (якщо є)
o	Дедлайн проєкту
6.	Форма додавання завдання:
•	Розташована під інформацією про проєкт
•	Поля:
o	Task Name — назва завдання
o	Deadline — кінцева дата виконання завдання
Кнопка Add Task для додавання нового завдання
7.	Таблиця завдань:
•	Структура таблиці:
o	Назва завдання
o	Дедлайн
o	Статус (To Do / In Progress / Done)
o	Кнопки Edit і Delete
При кліку на статус він змінюється циклічно між To Do / In Progress / Done
8.	Кнопки дій:
•	Add Project — створює новий проєкт
•	Add Task — додає нове завдання до вибраного проєкту
•	Edit — редагує проєкт або завдання
•	Delete — видаляє проєкт або завдання після підтвердження користувача
Поля введення відображені на Рисунку 5:
•	Project Name
o	Тип: текст
o	Обов’язкове поле: так
o	Перевірка: не можна залишати порожнім
•	Description
o	Тип: текст
o	Обов’язкове поле: ні
•	Project Deadline
o	Тип: дата
o	Обов’язкове поле: ні
•	Task Name
o	Тип: текст
o	Обов’язкове поле: так
o	Перевірка: не можна залишати порожнім
•	Task Deadline
o	Тип: дата
o	Обов’язкове поле: так
